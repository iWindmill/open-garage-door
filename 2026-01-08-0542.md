## Какие знания понадобились?

- map. `map(function, iterable, ...)`[^2]. Возвращает итератор, который применяет функцию к каждому элементу итерируемого объекта, выдавая результат. Если передаются дополнительные аргументы итерируемого объекта, функция должна принимать соответствующее количество аргументов и применяется к элементам всех итерируемых объектов параллельно. При наличии нескольких итерируемых объектов итератор останавливается, когда исчерпан кратчайший итерируемый объект. Например[^1], при вводе в одной строке данных через пробел `10 5 6` будет получен результат `21`:
	```python
	a, b, c = map(int, input().split())
	print(a + b + c)
	```
- ValueError[^2], exception. Эта ошибка возникает, когда операция или функция получает аргумент, имеющий правильный тип, но неподходящее значение, и ситуация не описывается более точным исключением, таким как IndexError. Например, вводе валидного типа данных `3 4 5` приведет к `ValueError: invalid literal for int() with base 10: '3 4 5'`, так как у нас неподходящее количество значений ожидаемых кодом:
	```python
	a = int(input())
	b = int(input())
	c = int(input())
	print(a + b + c)
	```

## Что понял?

Грустно, по передо мной очередной курс в котором сначала по завершении урока даются задача "напишите программу, которая считывает два числа и затем складывает их". А затем в следующем уроке встречается абзац "рассмотрим простую программу, которая считывает два числа и затем складывает их". Это похоже стиль такой - располагаешь задачи в уроках, а затем сталкиваешься с необходимостью расположить между уроками тесты и проверочные задачи. И просто берешь и задаешь упражнение, которое представляем задачу рассматриваемую в следующем уроке[^1]. В итоге тебе как создателю курса - легко. А как ученик потом осознаешь несоответствие между уровнем полученного материала и сутью задач. Так правильно же! Их понадергали из следующих уроков.
